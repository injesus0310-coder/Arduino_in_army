📝 [DevLog] Project: 3색 LED & 가변저항 제어 구현기
📅 날짜: 2026년 3월 31일 - 4월 2일 (D-201)
"군 생활 중 사지방 연등을 통해 하드웨어 제어 로직의 기초를 다지다."

1. ⚠️ 주요 트러블슈팅 (Troubleshooting)
① 전압 쇼트 및 회로 구성 오류

문제: 처음 회로를 구성했을 때 LED가 제대로 작동하지 않고 쇼트 현상이 발생함.
원인: 3색 LED의 공통 단자(Common Pin)와 각 색상 핀의 연결 관계를 혼동하여 배선함.
해결: 각 핀에 적절한 저항을 배치하고, 브레드보드의 흐름을 다시 파악하여 전류가 역류하거나 과부하가 걸리지 않도록 재배치함.

② =와 == 연산자 혼동 (Logic Error)

문제: 스위치를 눌러도 색상(color) 모드가 변경되지 않고 특정 상태에 고정됨.
원인: if(color = 0)과 같이 대입 연산자(=)를 비교문 안에 사용하여, 조건 체크가 아닌 값의 강제 할당이 일어남.
해결: 비교 연산자(==)로 수정하여 변수의 상태를 정확히 체크하도록 로직을 교정함.

③ 값의 대입 방향 오류 (Data Flow)

문제: 가변저항(Potentiometer)을 돌려도 LED의 밝기나 색상이 변하지 않음.
원인: pot = R_value;와 같이 읽어온 센서 값을 저장 변수에 넣는 것이 아니라, 초기값(0)을 센서 값 변수에 덮어씌우는 실수를 함.
해결: R_value = pot; 순서로 수정하여 읽어온 map 데이터가 실제 출력 변수에 저장되도록 수정함.

2. 💡 새로 배운 핵심 개념 (Key Learnings)
Potentiometer (가변저항): 전압의 크기를 조절하여 아날로그 값을 입력받는 부품. 공학적 명칭인 'Potentiometer'와 줄임말 'Pot'을 익힘.

Serial.begin(9600): 아두이노와 PC 간의 통신 시동을 거는 핵심 명령어. 이 코드가 빠지면 시리얼 모니터에 어떤 출력도 나오지 않음을 실전에서 경험함.

AnalogWrite & PWM: 디지털 핀(9, 10, 11번)을 통해 0~255 사이의 아날로그 전압 효과를 내는 원리를 이해함.]
